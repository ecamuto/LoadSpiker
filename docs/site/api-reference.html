API Reference

Complete reference for all LoadSpiker Python classes and methods.

Engine

The core engine class that manages the C-based connection pool and worker thread pool. Import from loadspiker.

Engine(max_connections=100, worker_threads=4, timeout_ms=30000, verbose=False)
ParameterTypeDefaultDescription
max_connectionsint100Maximum concurrent connections in the pool
worker_threadsint4Number of worker threads for request processing
timeout_msint30000Default request timeout in milliseconds
verboseboolFalseEnable verbose logging from the C engine

execute_request()

engine.execute_request(url, method="GET", headers=None, body=None, timeout_ms=None)

Execute a single HTTP request and return the response.

ParameterTypeDescription
urlstrTarget URL (must include protocol)
methodstrHTTP method: GET, POST, PUT, DELETE, PATCH, HEAD
headersdictOptional request headers as key-value pairs
bodystrOptional request body string
timeout_msintOverride the engine's default timeout

Returns: dict with keys status_code, body, headers, response_time_ms, error

response = engine.execute_request(
    "https://api.example.com/users/1",
    method="GET",
    headers={"Authorization": "Bearer my-token"}
)
print(response['status_code'])     # 200
print(response['response_time_ms']) # 142.3

run_scenario()

engine.run_scenario(scenario, users=1, duration=60, ramp_up=0)

Run a scenario with the specified load parameters. Releases the Python GIL during execution for true concurrency.

ParameterTypeDescription
scenarioScenarioThe scenario object to execute
usersintNumber of concurrent virtual users
durationintTest duration in seconds
ramp_upintSeconds to linearly ramp up to full user count
 
results = engine.run_scenario(
    scenario,
    users=100,
    duration=120,
    ramp_up=30
)
print(f"RPS: {results['requests_per_second']:.2f}")
print(f"Error rate: {results['error_rate']:.2%}")

get_metrics()

engine.get_metrics()

Retrieve the current accumulated metrics from the engine. Can be called mid-test for live monitoring.

KeyTypeDescription
total_requestsintTotal number of requests made
successful_requestsintRequests with 2xx status codes
failed_requestsintRequests with non-2xx or errors
requests_per_secondfloatThroughput over the test window
avg_response_time_msfloatMean response time in milliseconds
min_response_time_msfloatMinimum observed response time
max_response_time_msfloatMaximum observed response time
total_bytes_transferredintTotal bytes sent and received
error_ratefloatFraction of failed requests (0.0–1.0)
active_connectionsintCurrent open connections

reset_metrics()

engine.reset_metrics()

Reset all accumulated metrics counters to zero. Useful for baseline measurements or between test phases.

# Warm-up phase
engine.run_scenario(warmup_scenario, users=5, duration=10)
engine.reset_metrics()

# Actual measurement phase
results = engine.run_scenario(main_scenario, users=100, duration=60)

Database Methods

engine.database_connect(connection_string, pool_size=10)

Create a simulated database connection pool. Returns a connection ID string.

conn_id = engine.database_connect("mysql://user:pass@localhost/mydb", pool_size=20)
engine.database_query(connection_id, query, params=None)

Execute a query against the simulated connection pool. Returns a response dict with status_code, body, and response_time_ms.

result = engine.database_query(conn_id, "SELECT * FROM users WHERE id = ?", [42])
engine.database_disconnect(connection_id)

Release all connections in the pool and free associated resources.

WebSocket Methods

engine.websocket_connect(url, headers=None)

Initiate a WebSocket connection (simulated in Phase 1). Returns a connection ID string.

engine.websocket_send(connection_id, message)

Send a message over the WebSocket connection.

engine.websocket_close(connection_id)

Close the WebSocket connection and release resources.

ws_id = engine.websocket_connect("wss://echo.example.com/ws")
engine.websocket_send(ws_id, '{"type": "ping"}')
engine.websocket_close(ws_id)

MQTT Methods

engine.mqtt_connect(broker_host, broker_port=1883, client_id=None, username=None, password=None, keepalive=60)

Connect to an MQTT broker. Returns a connection ID string.

engine.mqtt_publish(connection_id, topic, payload, qos=0, retain=False)

Publish a message to a topic.

engine.mqtt_subscribe(connection_id, topic, qos=0)

Subscribe to a topic pattern.

engine.mqtt_disconnect(connection_id)

Disconnect from the MQTT broker.

mqtt_id = engine.mqtt_connect("mqtt.example.com", 1883,
                               client_id="loadspiker-1",
                               username="user", password="secret")
engine.mqtt_publish(mqtt_id, "sensors/temperature", "23.5", qos=1)
engine.mqtt_subscribe(mqtt_id, "alerts/#", qos=0)
engine.mqtt_disconnect(mqtt_id)

TCP Methods

engine.tcp_connect(host, port, timeout_ms=None)

Open a raw TCP connection. Returns a connection ID string.

engine.tcp_send(connection_id, data)

Send raw bytes or a string over the TCP connection.

engine.tcp_receive(connection_id, buffer_size=4096)

Receive data from the TCP connection. Returns received bytes.

engine.tcp_disconnect(connection_id)

Close the TCP connection.

tcp_id = engine.tcp_connect("tcp.example.com", 9000)
engine.tcp_send(tcp_id, "HELLO\r\n")
data = engine.tcp_receive(tcp_id)
engine.tcp_disconnect(tcp_id)

UDP Methods

engine.udp_create_endpoint(local_port=0)

Create a UDP endpoint bound to a local port (0 = OS-assigned). Returns an endpoint ID string.

engine.udp_send(endpoint_id, host, port, data)

Send a UDP datagram to the specified host and port.

engine.udp_receive(endpoint_id, buffer_size=4096, timeout_ms=1000)

Receive a UDP datagram. Returns a dict with data, source_host, source_port.

engine.udp_close_endpoint(endpoint_id)

Close the UDP endpoint and release the socket.

ep_id = engine.udp_create_endpoint()
engine.udp_send(ep_id, "udp.example.com", 5005, "PING")
response = engine.udp_receive(ep_id, timeout_ms=500)
print(response['data'])
engine.udp_close_endpoint(ep_id)

Scenario

A Scenario is a collection of requests that virtual users execute sequentially. Import from loadspiker.

Scenario(name, base_url=None)
ParameterTypeDescription
namestrHuman-readable name for this scenario
base_urlstrOptional base URL prepended to all relative request paths

Core Methods

scenario.get(url, headers=None, timeout_ms=None)

Add a GET request to the scenario.

scenario.post(url, body=None, headers=None, timeout_ms=None)

Add a POST request to the scenario.

scenario.put(url, body=None, headers=None, timeout_ms=None)

Add a PUT request to the scenario.

scenario.delete(url, headers=None, timeout_ms=None)

Add a DELETE request to the scenario.

scenario.add_request(url, method="GET", headers=None, body=None, timeout_ms=None)

Generic method for adding any HTTP request.

scenario.load_data_file(filepath, strategy="sequential")

Load CSV data for data-driven testing. Strategies: "sequential", "random", "round_robin".

scenario = Scenario("API Test", base_url="https://api.example.com")
scenario.load_data_file("users.csv", strategy="round_robin")
scenario.get("/users/{user_id}")  # {user_id} replaced from CSV data
scenario.post("/orders", body='{"product": "{product_id}", "qty": "{qty}"}')

RESTAPIScenario

A Scenario subclass pre-configured for JSON REST APIs.

from loadspiker import RESTAPIScenario

api = RESTAPIScenario("REST API Test", base_url="https://api.example.com")
# Automatically sets Content-Type: application/json
api.get("/users")
api.post("/users", body='{"name": "Alice"}')
api.put("/users/1", body='{"name": "Alice Updated"}')
api.delete("/users/1")

WebsiteScenario

A Scenario subclass simulating a user browsing a website.

from loadspiker import WebsiteScenario

site = WebsiteScenario("Website Browse", base_url="https://www.example.com")
site.visit("/")
site.visit("/about")
site.visit("/products")
site.think_time(2.0)  # Simulate 2-second pause between pages
site.visit("/contact")

MQTTScenario

from loadspiker.scenarios import MQTTScenario

mqtt_scenario = MQTTScenario("IoT Publish Test",
                              broker_host="mqtt.example.com",
                              broker_port=1883)
mqtt_scenario.connect(client_id="sensor-{user_id}", username="iot", password="secret")
mqtt_scenario.publish("sensors/{user_id}/temperature", payload="22.5", qos=0)
mqtt_scenario.publish("sensors/{user_id}/humidity", payload="65", qos=1)
mqtt_scenario.disconnect()

Reporters

Reporters format and output test results. Import from loadspiker.reporters.

ConsoleReporter

Prints a formatted summary table to stdout.

ConsoleReporter(color=True, show_progress=True)

JSONReporter

Writes results as JSON to a file or stdout.

JSONReporter(filepath=None, pretty=True)

HTMLReporter

Generates a self-contained HTML report with charts.

HTMLReporter(filepath, title="Load Test Report")

MultiReporter

Combines multiple reporters, dispatching to each.

MultiReporter(*reporters)

Reporter Interface

MethodDescription
start_reporting(scenario_name)Called before the test begins
report_metrics(metrics)Called periodically during the test with current metrics
end_reporting(final_results)Called after the test completes with final aggregated results

Complete Reporter Example

from loadspiker import Engine, Scenario
from loadspiker.reporters import ConsoleReporter, HTMLReporter, MultiReporter

engine = Engine(max_connections=200, worker_threads=8)
scenario = Scenario("Full Report Demo")
scenario.get("https://httpbin.org/get")

reporter = MultiReporter(
    ConsoleReporter(color=True),
    HTMLReporter("report.html", title="My Load Test"),
)

reporter.start_reporting("Full Report Demo")
results = engine.run_scenario(scenario, users=50, duration=60)
reporter.report_metrics(results)
reporter.end_reporting(results)

Utilities

Helper functions for common load patterns. Import from loadspiker.utils.

ramp_up(engine, scenario, start_users, end_users, ramp_duration, hold_duration=0)

Linearly increase load from start_users to end_users over ramp_duration seconds, then optionally hold.

from loadspiker.utils import ramp_up
results = ramp_up(engine, scenario,
                  start_users=5, end_users=100,
                  ramp_duration=60, hold_duration=120)
spike_test(engine, scenario, base_users, spike_users, spike_duration, cycles=3)

Alternate between baseline and spike load for the specified number of cycles.

from loadspiker.utils import spike_test
results = spike_test(engine, scenario,
                     base_users=10, spike_users=200,
                     spike_duration=15, cycles=5)
stress_test(engine, scenario, max_users, step_size=10, step_duration=30)

Incrementally increase load by step_size users every step_duration seconds until max_users is reached or errors exceed threshold.

constant_load(engine, scenario, users, duration)

Run a simple constant-load test. Equivalent to engine.run_scenario with no ramp-up.

parse_load_pattern(pattern_string)

Parse a CLI-style load pattern string into a dict of parameters. See CLI Load Pattern Syntax.

from loadspiker.utils import parse_load_pattern
params = parse_load_pattern("ramp:10:100:60")
# {'type': 'ramp', 'start_users': 10, 'end_users': 100, 'duration': 60}
