CLI Reference

Complete reference for the loadspiker command-line interface.

Synopsis

loadspiker [URL] [OPTIONS]

# Or using Python module invocation:
python3 -m loadspiker [URL] [OPTIONS]
loadspiker [-h] [--version]
           [-u USERS] [-d DURATION] [-r RAMP_UP] [-p PATTERN]
           [--max-connections MAX_CONNECTIONS] [--threads THREADS]
           [-m METHOD] [-H HEADER] [-b BODY] [-t TIMEOUT]
           [-s SCENARIO] [-c CONFIG] [-i]
           [--json [FILE]] [--html [FILE]] [-q] [--no-progress]
           [URL]

Positional Arguments

ArgumentDescription
URL Target URL to load test. Must include the protocol (e.g., https://example.com). Optional if --scenario is provided.

Load Parameters

FlagDefaultDescription
-u, --users 10 Number of concurrent virtual users
-d, --duration 60 Test duration in seconds
-r, --ramp-up 0 Seconds to linearly ramp from 0 to full user count before the main test window
-p, --pattern — Load pattern string (see Load Pattern Syntax). Overrides --users and --duration.

Engine Configuration

FlagDefaultDescription
--max-connections 100 Maximum simultaneous connections in the C engine connection pool
--threads 4 Number of worker threads in the C engine thread pool

Request Options

FlagDefaultDescription
-m, --method GET HTTP method: GET, POST, PUT, DELETE, PATCH, HEAD
-H, --header — Add a request header in Name: Value format. Repeatable.
-b, --body — Request body string. Use @filename to read from a file.
-t, --timeout 30000 Per-request timeout in milliseconds

Input Sources

FlagDescription
-s, --scenario Path to a Python scenario file. If provided, the URL positional argument is not required.
-c, --config Path to a JSON configuration file. Values here serve as defaults, overridden by flags.
-i, --interactive Launch the interactive REPL (see Interactive Mode).

Output Options

FlagDescription
--json [FILE] Write results as JSON. If FILE is omitted, outputs to stdout.
--html [FILE] Write an HTML report. Defaults to report.html if FILE is omitted.
-q, --quiet Suppress all output except fatal errors.
--no-progress Disable the live progress bar (useful in CI environments).

Load Pattern Syntax

The --pattern flag accepts a colon-delimited string describing the load shape.

constant

--pattern "constant:USERS:DURATION"
# Example: 50 users for 120 seconds
loadspiker https://example.com --pattern "constant:50:120"

ramp

--pattern "ramp:START_USERS:END_USERS:DURATION"
# Example: ramp from 5 to 100 users over 90 seconds
loadspiker https://example.com --pattern "ramp:5:100:90"

spike

--pattern "spike:BASE_USERS:SPIKE_USERS:SPIKE_DURATION:BASE_DURATION"
# Example: base of 10, spike to 300 for 15s, then back to 10 for 45s
loadspiker https://example.com --pattern "spike:10:300:15:45"

Examples

# Basic test: 20 users for 60 seconds
loadspiker https://httpbin.org/get --users 20 --duration 60

# POST request with JSON body and auth header
loadspiker https://api.example.com/orders \
  --method POST \
  --header "Content-Type: application/json" \
  --header "Authorization: Bearer mytoken123" \
  --body '{"product_id": 42, "qty": 1}' \
  --users 10 --duration 30

# Ramp-up test: 0 → 100 users over 30s, hold for 60s
loadspiker https://api.example.com/search \
  --users 100 --duration 90 --ramp-up 30

# Spike pattern
loadspiker https://api.example.com/ \
  --pattern "spike:10:500:10:50"

# Run a scenario file with a config, output JSON and HTML
loadspiker \
  --scenario tests/smoke.py \
  --config config.json \
  --json results.json \
  --html report.html

# CI-friendly: no progress bar, exit code reflects pass/fail
loadspiker https://api.example.com/health \
  --users 5 --duration 10 \
  --no-progress --quiet --json

Interactive Mode

Start the REPL with loadspiker --interactive or loadspiker -i.

The interactive session gives you a live shell to build and run tests incrementally.

CommandDescription
get <URL>Add a GET request to the current scenario
post <URL> <body>Add a POST request with an optional body
run [users] [duration]Execute the current scenario with given parameters
metricsDisplay the current metrics from the last run
resetReset metrics counters and clear the current scenario
helpList all available interactive commands
quit / exitExit interactive mode
 
$ loadspiker --interactive
LoadSpiker Interactive Mode (type 'help' for commands)
> get https://httpbin.org/get
Added GET https://httpbin.org/get
> get https://httpbin.org/status/200
Added GET https://httpbin.org/status/200
> run 5 30
Running scenario with 5 users for 30 seconds...
Total Requests: 342
RPS: 11.40
Avg Response: 87.2 ms
Error Rate: 0.00%
> metrics
[displays current metrics table]
> quit
