Getting Started

Get LoadSpiker installed and running your first load test in under 10 minutes.

Requirements

Requirement Details
Operating SystemLinux, macOS, or Windows (WSL2 recommended)
Python3.7 or later
libcurlDevelopment headers required for HTTP support
CompilerGCC 7+ or Clang 6+
Build ToolGNU Make
pkg-configRequired for library detection

Ubuntu / Debian

sudo apt-get update
sudo apt-get install -y \
  python3 python3-dev python3-pip \
  libcurl4-openssl-dev \
  gcc make pkg-config \
  build-essential

CentOS / RHEL

sudo yum groupinstall "Development Tools"
sudo yum install -y \
  python3 python3-devel python3-pip \
  libcurl-devel \
  gcc make pkg-config

macOS

# Install Homebrew if not already installed
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install dependencies
brew install python pkg-config curl

Windows (WSL2)

# Enable WSL2 and install Ubuntu from Microsoft Store, then:
sudo apt-get update
sudo apt-get install -y python3 python3-dev python3-pip \
  libcurl4-openssl-dev gcc make pkg-config build-essential

Installation

1

Clone the repository

git clone https://github.com/loadspiker/loadspiker.git
cd loadspiker
2

Install system dependencies

make install-deps
3

Build the C engine and Python extension

make build
4

Install the Python package

make install
# Or install in editable/development mode:
pip install -e .
5

Verify the installation

python3 -c "import loadspiker; print('LoadSpiker', loadspiker.__version__)"
loadspiker --version

Your First Test

Create a file named my_test.py and paste the following:

from loadspiker import Engine, Scenario

# Create the engine with resource limits
engine = Engine(max_connections=100, worker_threads=4)

# Define your scenario
scenario = Scenario("My First Load Test")

# Add HTTP requests
scenario.get("https://httpbin.org/get")
scenario.post(
    "https://httpbin.org/post",
    body='{"message": "hello"}',
    headers={"Content-Type": "application/json"}
)

# Run the test: 10 concurrent users for 60 seconds
results = engine.run_scenario(scenario, users=10, duration=60)

# Print results
print(f"Total Requests:   {results['total_requests']}")
print(f"Successful:       {results['successful_requests']}")
print(f"RPS:              {results['requests_per_second']:.2f}")
print(f"Avg Response:     {results['avg_response_time_ms']:.2f} ms")
print(f"Min Response:     {results['min_response_time_ms']:.2f} ms")
print(f"Max Response:     {results['max_response_time_ms']:.2f} ms")
print(f"Error Rate:       {results['error_rate']:.2%}")

# Clean up
engine.destroy()

Run it:

python3 my_test.py

CLI Quick Start

LoadSpiker also ships with a command-line interface for quick, scriptable tests.

# Quick test of a single URL
loadspiker https://httpbin.org/get --users 10 --duration 30

# With ramp-up period
loadspiker https://httpbin.org/get --users 50 --duration 60 --ramp-up 15

# Interactive mode (REPL)
loadspiker --interactive

# Generate an HTML report
loadspiker https://httpbin.org/get --users 20 --duration 30 --html report.html

Configuration Files

For repeatable tests, store configuration in JSON and scenario logic in Python.

JSON Config (config.json)

{
  "engine": {
    "max_connections": 200,
    "worker_threads": 8,
    "timeout_ms": 30000
  },
  "load": {
    "users": 50,
    "duration": 120,
    "ramp_up": 20
  },
  "reporting": {
    "console": true,
    "json": "results.json",
    "html": "report.html"
  }
}

Python Scenario File (scenario.py)

from loadspiker import Scenario

scenario = Scenario("API Smoke Test")

scenario.get(
    "https://api.example.com/health",
    headers={"Accept": "application/json"}
)

scenario.post(
    "https://api.example.com/users",
    body='{"name": "Test User", "email": "test@example.com"}',
    headers={"Content-Type": "application/json", "Authorization": "Bearer token123"}
)

Run using both files:

loadspiker --scenario scenario.py --config config.json

Target Description

make debugBuild with debug symbols and verbose logging
make cleanRemove all build artifacts
make dev-setupFull developer setup: deps + build + install + test
make packageCreate a distributable package archive
make benchmarkRun the built-in performance benchmarks
