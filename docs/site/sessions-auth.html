Sessions & Authentication

Manage per-user session state and authenticate against protected APIs during load tests.

Overview

LoadSpiker provides two complementary subsystems for stateful testing:

  • SessionManager — Thread-safe per-user session storage. Tracks cookies, tokens, and arbitrary key-value state across requests in a scenario run.
  • AuthenticationManager — Pluggable authentication flows. Supports Basic Auth, Bearer tokens, API Keys, Form-based login, OAuth 2.0, and custom flows.
Import paths:
  • from loadspiker.session_manager import get_session_manager, SessionStore
  • from loadspiker.authentication import get_authentication_manager

Session Management

The SessionManager is a singleton that provides a separate SessionStore per virtual user (keyed by user ID). All operations are thread-safe.

from loadspiker.session_manager import get_session_manager

sm = get_session_manager()  # Returns the singleton instance

Core SessionManager Methods

sm.get_session(user_id)

Get or create a SessionStore for the given user ID. Creates a new store if none exists.

sm.prepare_request_headers(user_id, base_headers=None)

Returns a headers dict merged with the session's stored authentication headers and cookies. Use this before every authenticated request.

sm.auto_handle_cookies(user_id, response)

Parse Set-Cookie headers from a response and store them in the user's session. Subsequent requests via prepare_request_headers will include these cookies.

sm.process_response(user_id, response, extract_rules=None)

Process a response: auto-handle cookies and optionally extract values using extract_rules. See Request Correlation.

from loadspiker import Engine
from loadspiker.session_manager import get_session_manager

engine = Engine()
sm = get_session_manager()
user_id = "user-001"

# Login request — cookies are extracted automatically
login_response = engine.execute_request(
    "https://app.example.com/login",
    method="POST",
    body='{"username": "alice", "password": "secret"}',
    headers={"Content-Type": "application/json"}
)
sm.process_response(user_id, login_response)

# Subsequent requests automatically include the session cookie
headers = sm.prepare_request_headers(user_id)
profile_response = engine.execute_request(
    "https://app.example.com/profile",
    headers=headers
)

SessionStore

Each user has an isolated SessionStore instance with the following methods:

store.set(key, value) / store.get(key, default=None)

Store and retrieve arbitrary session state.

store.set_token(token, token_type="Bearer") / store.get_token()

Store and retrieve an authentication token. get_token() returns a dict with token and token_type.

store.set_cookie(name, value, domain=None, path="/")
store.get_cookie(name) / store.get_all_cookies()

Manage cookies in the session store.

from loadspiker.session_manager import get_session_manager

sm = get_session_manager()
store = sm.get_session("user-42")

# Store arbitrary state
store.set("cart_id", "abc-123")
store.set("user_tier", "premium")

cart_id = store.get("cart_id")

# Store a JWT token
store.set_token("eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", token_type="Bearer")
token_info = store.get_token()

# Manual cookie management
store.set_cookie("session_id", "s-xyz-456", domain="app.example.com")
session_cookie = store.get_cookie("session_id")
all_cookies = store.get_all_cookies()

Request Correlation

Extract values from responses and store them in the session using extraction rules.

Rule TypeFieldDescription
json_pathpath, store_asExtract a value from a JSON response body using dot-notation path
headerheader_name, store_asExtract a response header value
cookiecookie_name, store_asExtract a cookie value from Set-Cookie headers
regexpattern, group, store_asExtract using a regular expression on the response body
 
from loadspiker import Engine
from loadspiker.session_manager import get_session_manager

engine = Engine()
sm = get_session_manager()
user_id = "user-001"

# Login and extract the JWT token from the response body
login_response = engine.execute_request(
    "https://api.example.com/auth/login",
    method="POST",
    body='{"username": "alice", "password": "secret"}',
    headers={"Content-Type": "application/json"}
)

sm.process_response(user_id, login_response, extract_rules=[
    {"type": "json_path",  "path": "token",        "store_as": "auth_token"},
    {"type": "json_path",  "path": "user.id",       "store_as": "user_id"},
    {"type": "header",     "header_name": "X-Req-Id", "store_as": "request_id"},
    {"type": "regex",      "pattern": r'"csrf":"([^"]+)"', "group": 1, "store_as": "csrf_token"},
])

store = sm.get_session(user_id)
token = store.get("auth_token")

# Use extracted token for subsequent requests
response = engine.execute_request(
    "https://api.example.com/orders",
    headers={
        "Authorization": f"Bearer {token}",
        "X-CSRF-Token": store.get("csrf_token"),
    }
)

Authentication

The AuthenticationManager singleton manages named authentication flows that can be applied to scenarios.

from loadspiker.authentication import get_authentication_manager

am = get_authentication_manager()  # Returns the singleton instance

AuthenticationManager Methods

am.register_flow(name, flow)

Register a named authentication flow for later use.

am.authenticate(flow_name, user_id, engine)

Execute the named flow for the given user. Stores resulting tokens/cookies in the user's SessionStore.

am.is_authenticated(flow_name, user_id)

Returns True if the user currently has valid authentication for the named flow.

am.logout(flow_name, user_id)

Invalidate and clear the user's authentication state for the named flow.

am.get_auth_headers(flow_name, user_id)

Returns the appropriate HTTP headers to authenticate a request for the named flow and user.

Basic Authentication

am.create_basic_auth(username, password)
from loadspiker import Engine
from loadspiker.authentication import get_authentication_manager

engine = Engine()
am = get_authentication_manager()

basic_flow = am.create_basic_auth(username="admin", password="secret")
am.register_flow("basic", basic_flow)

am.authenticate("basic", "user-001", engine)

headers = am.get_auth_headers("basic", "user-001")
# headers = {"Authorization": "Basic YWRtaW46c2VjcmV0"}
response = engine.execute_request("https://api.example.com/admin", headers=headers)

Bearer Token Authentication

am.create_bearer_auth(token=None, token_endpoint=None, client_id=None, client_secret=None)
from loadspiker.authentication import get_authentication_manager

am = get_authentication_manager()

# Static token
bearer_flow = am.create_bearer_auth(token="eyJhbGciOiJIUzI1NiJ9...")
am.register_flow("bearer", bearer_flow)

# Dynamic token from endpoint (fetched on first authenticate() call)
dynamic_flow = am.create_bearer_auth(
    token_endpoint="https://auth.example.com/token",
    client_id="my-client",
    client_secret="my-secret"
)
am.register_flow("dynamic_bearer", dynamic_flow)

API Key Authentication

am.create_api_key_auth(api_key, header_name="X-API-Key", location="header")
from loadspiker.authentication import get_authentication_manager

am = get_authentication_manager()

# Header-based API key
api_key_flow = am.create_api_key_auth(
    api_key="sk-live-abc123",
    header_name="X-API-Key"
)
am.register_flow("apikey", api_key_flow)

# Custom header name (e.g. some APIs use "api-token")
custom_flow = am.create_api_key_auth(
    api_key="my-secret-key",
    header_name="api-token"
)
am.register_flow("custom_key", custom_flow)

headers = am.get_auth_headers("apikey", "user-001")
# headers = {"X-API-Key": "sk-live-abc123"}

Form-Based Authentication

am.create_form_auth(login_url, username, password, username_field="username", password_field="password", success_indicator=None)
from loadspiker import Engine
from loadspiker.authentication import get_authentication_manager

engine = Engine()
am = get_authentication_manager()

form_flow = am.create_form_auth(
    login_url="https://app.example.com/login",
    username="test_user",
    password="test_password",
    username_field="email",         # field name in the form
    password_field="password",
    success_indicator='"logged_in": true'  # string to look for in response
)
am.register_flow("form", form_flow)

# Authenticate (performs the login POST and stores session cookies)
am.authenticate("form", "user-001", engine)

# All cookies from the login response are now in the session
headers = am.get_auth_headers("form", "user-001")

OAuth 2.0

am.create_oauth2_auth(client_id, client_secret, auth_url=None, token_url=None, scope=None, grant_type="client_credentials")
from loadspiker import Engine
from loadspiker.authentication import get_authentication_manager

engine = Engine()
am = get_authentication_manager()

# Client Credentials flow (machine-to-machine)
oauth2_flow = am.create_oauth2_auth(
    client_id="my-app-client-id",
    client_secret="my-app-client-secret",
    token_url="https://auth.example.com/oauth/token",
    scope="read:users write:orders",
    grant_type="client_credentials"
)
am.register_flow("oauth2", oauth2_flow)

# Authenticate (fetches and caches the access token)
am.authenticate("oauth2", "user-001", engine)

headers = am.get_auth_headers("oauth2", "user-001")
# headers = {"Authorization": "Bearer eyJhbGciOiJSUzI1NiJ9..."}

Custom Authentication

am.create_custom_auth(auth_function)

Provide a callable auth_function(user_id, engine, session_store) that performs authentication and populates the session store with tokens or cookies.

import json
from loadspiker import Engine
from loadspiker.authentication import get_authentication_manager

engine = Engine()
am = get_authentication_manager()

def my_auth_fn(user_id, engine, session_store):
    """Custom two-step authentication flow."""
    # Step 1: Get CSRF token from the login page
    csrf_response = engine.execute_request("https://app.example.com/login")
    import re
    csrf_match = re.search(r'name="csrf_token" value="([^"]+)"', csrf_response['body'])
    csrf_token = csrf_match.group(1) if csrf_match else ""

    # Step 2: Submit login form with CSRF token
    login_response = engine.execute_request(
        "https://app.example.com/login",
        method="POST",
        body=f"username=testuser&password=secret&csrf_token={csrf_token}",
        headers={"Content-Type": "application/x-www-form-urlencoded"}
    )

    # Extract and store session cookie
    session_cookie = login_response['headers'].get('Set-Cookie', '')
    session_store.set_cookie("session", session_cookie)
    session_store.set("authenticated", True)

custom_flow = am.create_custom_auth(my_auth_fn)
am.register_flow("custom", custom_flow)
am.authenticate("custom", "user-001", engine)

Multi-User Session Isolation

Each virtual user maintains completely isolated session state. This is essential for realistic load tests simulating many concurrent users.

from loadspiker import Engine
from loadspiker.session_manager import get_session_manager
from loadspiker.authentication import get_authentication_manager

engine = Engine(max_connections=50)
sm = get_session_manager()
am = get_authentication_manager()

users = [
    {"id": "alice",   "username": "alice",   "password": "pass_alice"},
    {"id": "bob",     "username": "bob",     "password": "pass_bob"},
    {"id": "charlie", "username": "charlie", "password": "pass_charlie"},
]

for user in users:
    # Each user gets their own session store
    flow = am.create_form_auth(
        login_url="https://app.example.com/login",
        username=user["username"],
        password=user["password"]
    )
    flow_name = f"form_{user['id']}"
    am.register_flow(flow_name, flow)
    am.authenticate(flow_name, user["id"], engine)

    store = sm.get_session(user["id"])
    print(f"{user['id']} cookies: {store.get_all_cookies()}")

# Run scenario with per-user sessions
from loadspiker import Scenario
scenario = Scenario("Multi-User Test")
scenario.get("https://app.example.com/dashboard")

results = engine.run_scenario(scenario, users=len(users), duration=30)

Best Practices

  • Authenticate before the measurement window: Run a warm-up or setup phase to authenticate all virtual users before starting timed measurements. Use engine.reset_metrics() after warm-up.
  • Use unique user IDs per virtual user: Pass a string like f"user-{thread_id}" as the user_id to ensure complete session isolation across concurrent threads.
  • Handle token expiry: Check am.is_authenticated() periodically and call am.authenticate() again if tokens have expired during long-running tests.
  • Clean up after tests: Call am.logout() to clean up server-side sessions at the end of a test run, especially during integration testing against real backends.
  • Avoid sharing session stores: Never share a SessionStore object between virtual users — this defeats the purpose of isolation and will cause data races.
  • Store minimal state: Only persist what is needed for correlation. Overly large session stores (e.g., storing full response bodies) will increase memory usage under high user counts.
  • Rotate credentials: For large-scale tests, provision a pool of test users with separate credentials. Reuse of a single credential across all virtual users may trigger server-side rate limits or lock-outs.
